## Memory — Project Continuity

Use memory for narrative context and project history.

**Source of truth:**
- `qt`, `qp`, `qr` CLI tools are **canonical** for task state, project status, and roadmaps
- Memory provides **context** — why decisions were made, what was tried, session narratives
- Never update task/project state based on memory alone — verify with the CLI tools first

**What to search for:**
- Prior session context when resuming work on a project
- Decision rationale — "why did we choose X over Y?"
- Handoff notes and plan documents from previous sessions
- Historical blockers and how they were resolved

**Useful collections:**
- `sessions` — past session logs and narratives
- `plans` — planning documents and architecture decisions
- `handoffs` — context transfer documents between sessions

## Memory — Analysis

Search for prior decisions and rationale before proposing new abstractions.

**When to search:**
- Before recommending an architecture change — check if it was discussed or rejected before
- When analyzing trade-offs — find prior analysis on similar decisions
- When debugging a recurring issue — check if it's been investigated before
- When asked "why was this done this way?"

**What to look for:**
- Design decisions and their rationale in plans, sessions, and docs
- Previous trade-off analyses and their conclusions
- Patterns that were tried and abandoned (and why)
- Cross-cutting concerns that affected past decisions

**How to use results:**
- Cite prior decisions — "this was discussed in session X, where Y was chosen because Z"
- Flag contradictions — "the current proposal conflicts with the rationale in plan X"
- Build on prior work — don't re-derive conclusions that already exist
